# Shikaku Race Studio 要件定義

## 1. 目的

本アプリは、`Square Race` 形式の観戦動画を短時間で量産するためのブラウザベース制作ツールである。  
ユーザーがこのアプリで行う主作業は「遊ぶ」ことではなく、以下である。

- コースを自動生成する
- 出力サイズを選ぶ
- レースを走らせる
- 画面を録画して動画素材として保存する

想定公開先:

- YouTube Shorts
- Instagram 正方形投稿

## 2. 対象ユーザー

- Shorts / Reels / SNS 向けに視聴用レース動画を量産したい制作者
- コース差分を大量に試したい運用者
- Seed ベースで再現可能なレースを管理したい運用者

## 3. 機能要件

### 3.1 出力

- 出力プリセットは以下の 2 種類に限定する
- `720 x 1280` : YouTube Shorts 用
- `720 x 720` : 正方形投稿用

### 3.2 レース

- レーサー数は `2` から `6`
- レーサーは正方形であること
- レーサーは四角い生物のような可愛い見た目であること
- レーサーは初速を持って自動走行すること
- 全レーサーは同一の基準速度で走行すること
- レーサーは壁に衝突した際、反射して進行方向を変えること
- 壁に衝突した際、`ぽよん` とした変形アニメーションを表示すること
- レーサー同士が接触した場合、壁衝突に準じる反発を互いに与えること
- 壁衝突時にはレーサーごとに異なる音が鳴ること
- 先頭の 1 台がゴールした時点でレース終了とする
- タイムアップは設けない

### 3.3 コース生成

- コースは `15 x 20` の固定グリッドで構成する
- 各レーサーには `2 x 2` マスの個別スタート枠を用意する
- ゴール地点は `2 x 2` マスの正方形とする
- ゴール地点の見た目は `4 x 4` の白黒チェッカーマークとする
- レーサーは通常セルより明確に小さいサイズで描画する
- レーサーは目や表情を持ち、識別番号に頼らず視認できること
- コースは「障害物を並べる」のではなく「通路を掘る」方式で生成する
- 通路は必ずスタートからゴールまで連結していること
- 通路幅は `1` マスから `3` マスの範囲で制御する
- 壁は煉瓦模様で描画する
- 通路パラメータは以下とする
- `道の長さ`
- `道幅モード`
- `Seed`
- `コーススタイル`

### 3.4 録画

- キャンバス映像を録画できること
- 可能な環境では MP4 で保存すること
- MP4 非対応環境では WebM へフォールバックすること

### 3.5 表示切替と勝者演出

- 上部タイトルフレームと下部順位フレームは設定で ON/OFF できること
- 最初の 1 台がゴールした場合、勝者が明確にわかる派手な演出を表示すること
- 勝者演出は約 `5s` 表示すること
- 自動連戦が ON の場合、勝者演出後に次レースへ進むこと

### 3.6 再現性

- Seed が同じであれば同一コースを再生成できること
- コース設定は JSON として出力・再読込できること

## 4. 非機能要件

- 動画上で見やすいことを最優先とする
- UI は制作補助に徹し、画面上の情報がコースを隠しすぎないこと
- 通路と壁の判別が一目でつく配色であること
- 生成結果は毎回大きく変わり、動画素材として差分が出ること

## 5. コース作成の詳細定義

### 5.1 基本構造

- コースは `15 x 20` セルの二値グリッドで表現する
- 各セルは `通路` または `壁` のどちらかである
- スタートとゴールは通路セルとして扱う
- レーサーが走行可能なのは通路セルの内部のみである

### 5.2 座標ルール

- 左上を `(0, 0)` とする
- 右方向に `x`、下方向に `y` が増加する
- スタートは左側に配置する
- ゴールは右側に配置する
- スタートとゴールは外周に接しすぎないよう `1` マス以上の余白を持つ

### 5.3 スタート / ゴール定義

- 各レーサーに対して `2 x 2` の個別スタート枠を 1 つずつ持つ
- スタート枠は左側に縦に並べて配置する
- スタート枠同士の間には最低 `1` マスの余白を設ける
- ゴールは `2 x 2` の正方形セル領域とする
- ゴール内部の見た目は `4 x 4` の白黒チェッカーとする
- スタート枠とゴールは通路セルとして扱う

### 5.4 主経路生成

主経路は「左側の複数スタート枠から合流し、その後に右へ進みながら上下へ迂回する」方式で生成する。

生成手順:

1. レーサー数に応じて左側に個別スタート枠を配置する
2. 各スタート枠から共通の合流点まで接続通路を作る
3. 合流点からゴールへ向かう主経路の中心線を作る
4. 中心線は以下の 2 種類の移動で構成する
5. `右へ進む水平移動`
6. `上または下へ迂回する垂直移動`
7. 最終的にゴール中心へ必ず到達する

### 5.5 道の長さ

`道の長さ` は、壁配置を考慮したスタートからゴールまでの最短到達距離である。

- 距離は通路セル上の移動ステップ数で評価する
- 距離計測は壁を避けて進む最短経路で行う
- 生成器は `最短到達距離 >= 指定した道の長さ` を満たすコースのみ採用する
- 設定値が大きいほど、上下の迂回量や折り返しが増える
- 設定値が小さいほど、より直線的な経路になる

### 5.6 道幅モード

`道幅モード` は、コース内で許可する通路幅帯を定義し、コース難易度を調整する。

- 低設定では `1` マス幅中心
- 中設定では `1` から `2` マス幅を混在
- 高設定では `1` から `3` マス幅を混在
- 狭い区間が多いほど、衝突や詰まりが起きやすくなる
- 幅変化は中心線上の各区間単位で行う

### 5.7 コーススタイル

スタイルは、主経路の性格を決めるプリセットである。

- `snake` : 右進行の合間に大きな上下スイングを入れる
- `ladder` : 横移動は短く、縦移動が長い
- `canyon` : 幅広で比較的なだらか
- `zigzag` : 幅が細く、細かい折り返しが多い
- `variety` : 上記からランダム選択

各スタイルは以下を内部パラメータとして持つ。

- 水平移動の最大長
- 垂直スイングの最小長
- 垂直スイングの最大長
- 通路幅の最小値
- 通路幅の基準値
- 通路幅の最大値
- 縦方向へ振る傾向
- 交互に上下へ振るかどうか

### 5.8 通路の彫り方

- 中心線の各点に対して、現在の通路幅 `1` から `3` で領域を掘る
- 隣接する中心線点の間は矩形接続して切れ目をなくす
- その結果、中心線は視覚的に連続した通路になる
- 各スタート枠から合流点までの接続通路も同じルールで掘る
- 掘られなかったセルは壁になる

### 5.9 妥当性条件

生成されたコースは以下を満たす必要がある。

- スタートからゴールまで通路が連結している
- 全スタート枠が主経路へ接続している
- 通路がグリッド外へはみ出さない
- 通路幅が設定されたモードの範囲に収まる
- スタート / ゴールが壁に埋まらない

### 5.10 ランダム生成条件の伝え方

コース生成を希望に近づけるには、以下の条件を言語化できると実装しやすい。

- `全体形`: 例 `縦に長い`, `中央で大きく蛇行`, `下側に寄る`
- `合流位置`: 例 `早めに合流`, `最後まで別ルート感を残す`
- `曲がりの強さ`: 例 `直線多め`, `折り返し多め`
- `細い道の比率`: 例 `1マス道を多く`, `2マス道を中心`
- `詰まりやすさ`: 例 `接触しやすい`, `比較的スムーズ`
- `ゴール前の演出`: 例 `最後だけ細くする`, `最後に大きく曲げる`

条件をまだ言語化できない場合は、参考画像やラフな例を複数提示すればよい。

- `3` から `5` 個ほどの例があると傾向を抽出しやすい
- 画像ごとに `好きな点` と `嫌な点` を一言添えるとさらに精度が上がる
- 例: `この画像は合流が遅いのが好き`, `これは道幅が太すぎるので嫌`

## 6. レース進行の詳細定義

### 6.1 壁衝突

- レーサーが壁セルへ重なった場合、侵入前の方向へ押し戻す
- 衝突軸の速度を反転する
- 反射係数は 1 未満とし、完全なエネルギー保存にはしない

### 6.2 コース外脱出防止

- 毎フレーム、レーサーが通路セル上に残っているかを判定する
- 判定は中心点だけでなく、四隅と中心を含む複数サンプルで行う
- 通路外に出た場合は、最後に正常だった位置へ戻す
- 戻したうえで速度ベクトルを反射させ、停止しにくくする

### 6.3 レーサー同士の衝突

- レーサー同士が重なった場合、位置補正で分離する
- 相対速度が小さくても、壁衝突に準じる反射ベクトルを互いに与える
- 完全に重なった場合でも、決定的な法線ベクトルを作って分離する
- 分離後は基準速度へ正規化し、密着停止を防ぐ

## 7. タイムアップの定義

- タイムアップは設けない
- 旧実装の `28s` 固定終了は廃止する
- 現行仕様では、最初の 1 台がゴールした時点でレース終了とする

## 8. 既知の改善方針

- コース形状をさらに「枝道あり迷路型」へ寄せる余地がある
- 主経路の見栄えをもっと縦動画向けに最適化できる
- ゴール手前の演出や追い上げ演出は今後追加可能
